# 强缓存

浏览器中的缓存作用分为两种情况，一种是需要发送HTTP请求，一种是不需要发送的。

首先是检查强缓存，这个阶段是不需要发送HTTP请求的。

如何检查缓存呢？ 在HTTP/1.0和HTTP/1.1中，字段是不一样的，在HTTP/1.0中，使用的是Expires，而HTTP/1.1使用的是Cache-Control。

# Expires

Expires即过期时间，存在于服务端返回的响应头中，告诉浏览器在这个过期时间之内可以直接从缓存中获取数据，无需再次请求。比如：

Expires: Wed, 22 Nov 2019 08:41:00 GMT

表示资源在19年11月22号8点41分过期。 这个方法隐藏了一个问题——服务器时间和浏览器时间并不一致，那服务器返回的过期时间就不准确。因此这个方式很快被HTTP/1.1版本抛弃。

# Cache-Control

HTTP/1.1版本

它与Expires本质的不同在于它没有才去具体过期时间这种方式，而是采用过期时长来控制缓存。

Cache-Control可取的字段有：

max-age：max-age=3600，表示在响应后3600秒内，使用强缓存。

public：客户端和代理服务器都可以缓存。一个请求可能经过不同代理服务器，最后才到目标服务器，那么结果不仅仅是浏览器可以缓存数据，中间任何节点都可以缓存。

private：只有浏览器可以缓存，中间的代理服务器不能缓存。

no-cache：跳过强缓存，发送HTTP请求，进入协商缓存阶段。

no-store：不进行任何缓存。

s-maxage：针对代理服务器的缓存时间。

当Expires和Cache-Control同时存在时，Cache-Control的优先级会高于Expires。

当强缓存过期后，会进入协商缓存阶段。

# 协商缓存

当强缓存失效后，浏览器会携带相应的缓存tag向服务器发请求，由服务器根据这个tag来决定是否使用缓存。

缓存tag有两种ETag和Last-Modified。

# Last-Modified

即最后修改时间。在浏览器第一次给服务器发送请求后，服务器会在响应头中加上这个字段。

浏览器接收到后，如果再次请求，会在请求头中携带If-Modified-Since字段，这个字段的值也就是之前服务器传来的Last-Modified的值。

服务器拿到If-Modified-Since后会与服务器中的该资源的最后修改时间对比：

  • 如果请求头中的If-Modified-Since字段的值小于该资源的最后修改时间，则表示资源已经更新，返回新的资源。
  • 否则返回304，告诉浏览器直接使用缓存

# ETag

ETag是服务器根据当前文件内容，给文件生产的唯一标识，只要内容有改动，这个值就会变。服务器通过响应头把这个值给浏览器。

浏览器接收到ETag的值，会在下次请求时，将这个值作为If-None-Match这个字段的值放到请求头，发送给服务器。

服务器拿到If-None-Match后，会与服务器的ETag对比：

  • 如果两者不一样，则资源已更新。
  • 否者返回304，告诉浏览器使用缓存。

# 两者对比

  1. 在精度上，ETag高于Last-Modified。
  2. 在性能上，Last-Modified优于ETag，Last-Modified仅记录一个时间，ETag根据文件生成哈希值。

如果二者字段都存在，服务器会优先考虑ETag。

# 缓存位置

当我们从缓存中获取资源时，是从什么地方获取到的呢？

浏览器中的缓存为止一共有4中，按优先级从高到低排为：

  • Service Worker
  • Memory Cache
  • Disk Cache
  • Push Cache

# Service Worker

Service Worker借鉴了Web Worker的思路，即让JS运行在主线程之外，由于它脱离了浏览器窗体，因此无法直接访问DOM。虽然如此，但它仍然能帮我们完成很多功能，如离线缓存、消息推送和网络代理等功能。其中离线缓存就是Service Worker Cache。

Service Worker 同时也是 PWA 的重要实现机制，关于它的细节和特性，我们将会在后面的 PWA 的分享中详细介绍。

# Memory Cache and Disk Cache

Memory Cache指的是内存缓存，效率高但是存活时间短。

Disk Cache存在磁盘中的缓存，效率低于Memory Cache，但在存储量和存储时长上优于Memory Cache。

资源存放策略：

  • 较大的js，css文件会被放进磁盘，反之则在内存。
  • 内存使用率较高时，文件优先放入磁盘。

# 总结

首先检验Cache-Control字段确认强缓存是否可用，如果可用则直接使用，返回200

强缓存失效则根据ETag或者Last-Modified在请求头中携带响应If-None-Match或者If-Modified-Since发送给服务器进行比对确定是否使用缓存，如果可用则返回304，否则返回新的资源和200状态码。
