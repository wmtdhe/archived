# 什么是XSS

XSS —— Cross Site Scripting，即跨站脚本。XSS攻击指的是浏览器中执行恶意脚本，从而拿到用户的信息并进行操作。

这些操作一般可以：

  1. 窃取Cookie
  2. 监听用户行为，比如输入账号密码后直接发送到黑客服务器。
  3. 修改DOM伪造登录表单
  4. 在页面中产生浮窗广告

通常XSS的实现由三种方法——存储型、反射型和文档型。

# 存储型

将脚本存储到了服务端的数据库，然后在客户端执行这些脚本时进行攻击。

# 反射型

非持久化。没有存储到数据库

Example:

<form>
    <input type='text' name='q'/>
    <input type='submit'/>
</form>

app.use((ctx)=>{
    let { q } = ctx.query;
    ctx.body = q
})

如果用户直接输入<script>alert(1)</script>而后端没有过滤的话，会直接弹出alert。

# 什么是CSRF攻击？

Cross-site request forgery——即跨站请求伪造，指的是黑客诱导用户点击链接，打开黑客的网站，然后黑客利用用户目前的登录状态发起跨站请求。

# 自动发送GET

比如在黑客网站中：

<img src='https://xxx.com/info?user=xx&count=10'>

当你访问这个网站时，就会自动发送这个请求到xxx.com，假定你已经在这个xxx.com登录了，黑客就会携带上你在这个网站上的cookie访问到xxx.com。

# 防范

  1. 利用Cookie的SameSite属性

SameSite有三个值可取，Strict, Lax, None。

在Strict模式下，浏览器完全禁止第三方请求携带Cookie。比如请求wmtdhe.com网站只能在wmtdhe.com域名中的请求才能携带Cookie。

Lax，宽松一点，只能在get方法或者a标签发送get请求时可以携带Cookie

None，默认模式，请求会自动带上Cookie

  1. 验证来源站点

通过请求头中的两个字段：Origin和Referer。

Origin只包含域名信息，而Referer包含了具体路径。

但是Ajax可以自定义伪造referer和origin，安全性较差。

  1. CSRF Token

当客户端向服务端请求时，服务端会生成一串CSRF Token返回给客户端，服务端会把这串token存在当前用户的session中，从别的网站发起的伪造请求是拿不到这串token的，因此会被服务端拒绝请求。
