# 1.JS原始数据类型与引用数据类型

原始数据类型：

  • number
  • null
  • undefined
  • string
  • symbol
  • boolean

基本类型存储在栈内存，存储的是值。

引用数据类型：

  • Object（包含普通对象Object，数组对象Array，正则对象RegExp，日期对象Date，数学对象Math，函数对象Function）

复杂类型地址存储在栈内存，值存储在堆内存。

# 2.输出下列运行结果，解释原因

function test(person){
    person.age = 26;
    person = {
        name: 'wmtdhe',
        age: 20
    }
    return person
}
const p1 = {
    name: 'aka'.
    age: 18
};
const p2 = test(p1);
console.log(p1); // --> ?
console.log(p2); // --> ?

运行结果：

p1: {name: 'aka', age: 26}
p2: {name: 'wmtdhe', age: 20}

在函数传参的时候传递的是对象在堆中的内存地址值，test函数中的实参person是p1对象的内存地址，通过调用person.age = 26确实改变了p1的值，但随后person变成了另外一个内存空间的地址，并在最后将这另外一份内存空间地址返回赋给了p2。

# 3. null是对象吗？

虽然typeof null返回的结果为object，但null是一个基本数据类型。

# typeof & instanceof，instanceof实现原理

typeof能够正确判断基本数据类型，除了typeof null返回值为object。

但typeof不能正确判断对象的类型，typeof 一个函数可以输出'function'，而除此之外，输出的全是'object'。

instanceof是通过原型链判断的，A instanceof B，在A的原型链中层层查找，是否有原型等于B.prototype，如果一直找到A的原型链等短(null,即Object.prototype.__proto__)，仍然不等于B.prototype,那么返回false。

# 手动实现instanceof

function myInstanceof(left,right){
    //基本类型
    if(typeof left !=='object' || left===null)return false;
    let proto = left.__proto__; //Object.getPrototypeOf(left) 也能够拿到参数的原型
    while(true){
        if(proto===null)return false;
        if(proto===right.prototype)return true;
        proto = proto.__proto__;
    }
}

console.log(myInstanceof('111',String)) // false
console.log(myInstanceof(new String('111'),String)) // true

# 4. 为什么 0.1+0.2不等于0.3

0.1和0.2在转换成二进制后会无限循环，由于标准位数的限制后面多余的位数会被截掉，此时就已经出现了精度的损失，相加后因浮点数小数位的限制而阶段的二进制数字在转换为十进制就会变成0.30000000000000004。

# 5.Object.is()和===的区别？

Object.is()在严格等于的基础上修复了一些特殊情况下的失误。具体来说为+0和-0，NaN和NaN。

function is(x,y){
   if(x === y){
       //+0, -0
       //1/+0 = +Infinity, 1/-0 = -Infinity
       return x!==0 || y!==0 || 1/x === 1/y;
   }else{
       //NaN===NaN为false，而Object.is(NaN,NaN)为true。因此 x!==x肯定为NaN，x和y都是NaN时return true
       return x!==x && y!==y
   }
}

# ==和===有什么区别？

===为严格相等于，不仅左右的值要相等，类型也要相等。

==不像===那样严格，对于一般情况，只要值相等就返回true。但==还涉及一些类型转换，转换规则如下：

  • 两边的类型是否相同，相同就比较大小
  • 判断是否是null和undefined，是的话就return true
  • 判断是否是String和Number，是的话就把String类型转换为Number再比较
  • 判断其中一方是否是Boolean，是的话就转化为Number再比较
  • 如果其中一方是Object，且另一方是String、Number或Symbol，会将Object转换为字符串，再进行比较
console.log({a:1} == true) //false
console.log({a:1} == '[object Object]') //true

# 6. []==![]的结果是什么？

==中，左右两边都需要转换为数字然后进行比较。

[]转换为0。

![]首先转换为布尔值，[]作为引用类型的布尔值为true，因此![]为false，进而转换为数字0。

[]==![] --> 0 == 0 --> true

# JS中类型转换有哪几种

有三种

  • 转换成数字
  • 转换为布尔值
  • 转换成字符串

# 7.对象转原始类型是根据什么流程运行的？

对象转原始类型，会调用内置的[ToPremitive]函数，对该函数而言，其逻辑如下：

  1. 如果Symbol.toPremitive(),优先调用返回
  2. 调用valueOf()如果转换为原始类型，则返回，
  3. 调用toString()，如果转化为原始类型则返回
  4. 如果都没有返回原始类型则报错
let obj = {
    value:3,
    valueOf(){
        return 4;
    },
    toString(){
        return '5';
    },
    [Symbol.toPremitive](){
        return 6
    }
}
console.log(obj + 1) // 7

# 如何让if(a == 1 && a==2 )条件成立

let a = {
    value:0,
    valueOf(){
        this.value++;
        return this.value;
    }
}
console.log(a==1 && a==2); // true
