# 闭包

# 什么是闭包

闭包是指有权访问另外一个函数作用域中的变量的函数

# 闭包产生的原因

ES5中只存在两种作用域 -- 全局作用域和函数作用域，当访问一个变量时，解释器会首先在当前作用域查找标识符，如果没有找到，就去父作用域，知道找到该变量的标识符或者不在父作用域中，这就是作用域链。每个子函数都会拷贝上级的作用域，形成一个作用域链，比如：

var a = 1;
function f1(){
    var a = 2;
    function f2(){
        var a = 3;
        console.log(a); // 3
    }
}

以上代码中，f1的作用是指向全局（window）和它本身，f2指向全局，f1和它本身。

闭包产生的本质就是，当前环境中存在指向父级作用域的引用。

function f1(){
    var a = 2;
    function f2(){
        console.log(a); // 2
    }
    return f2;
}
var x = f1();
x();

这里的x拿到了父级作用域中的变量，输出2。

# 闭包有哪些表现形式

  1. 返回一个函数(以上代码)
  2. 作为函数参数传递
var a = 1;
function foo(){
    var a = 2;
    function baz(){
        console.log(a)
    }
    bar(baz);
}

function bar(fn){
    //这就是闭包
    fn()
}
//输出 2
foo();
  1. 在定时器，事件监听，Ajax请求，跨窗口通信，Web Workers或者任何一部中，只要使用了回调函数，实际上就是在使用闭包。
//以下闭包保存的作用域为window和本身
setTimeout(function(){
    
},100)
  1. IIFE（立即执行函数表达式）创建闭包，保存了全局作用域window和当前函数的作用域，
var a = 2;
(function IIFE(){
    console.log(a); // 2
})();

# 如何解决下面的循环输出问题？

for(var i=1;i<=5;i++){
    setTimeout(function timer(){
       console.log(i); 
    },0)
}

因为setTimeout是宏任务，由于JS中的单线程eventLoop机制，在主线程同步任务执行完成后才去执行宏任务，因此循环结束后setTimeout中的回调才依次执行，单输出的i的时候当前作用域没有，往上一级找，发现了i，此时循环已经结束，i变成了6，因此会全部输出6。

解决办法：

  1. 利用IIFE，当每次for循环时，把此时的i变量传递到定时器中
for(var i=0;i<=5;i++){
    (function(j){
        setTimeout(function timer(){
            console.log(j)
        },0)
    })(i);
}
  1. 给定时器传入第三个参数，作为timer函数的第一次参数
for(var i=0;i<=5;i++){
    setTimeout(function timer(j){
        console.log(i)
    },0,i)
}
  1. 使用ES6中的let
for(let i=0;i<=5;i++){
    setTimeout(function timer(){
        console.log(i);
    },0)
}

let使JS发生革命性的变化，让JS由函数作用域变为块级作用域，用let后作用域链不复存在。代码的作用域以块级为单位，以上代码为例。

// i = 1
{
    setTimeout(function timer(){
        console.log(1)
    })
}
// i = 2
{
    setTimeout(function timer(){
        console.log(2)
    })
}
// i = 3
