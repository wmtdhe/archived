# call实现

首先：

function showProfileMessage(message){
    console.log(message,this.name)
}

const obj = {
    name: 'wmtdhe'
};

showProfileMessage.call(obj,'welcome');// welcome wmtdhe

让我们分析一下call做了什么：

  1. 调用call方法改变了this的指向，上面的函数调用变成了obj.showProfileMessage
  2. 无论我们给showProfileMessage.call传了什么参数进去，都应该传给原来的showProfileMessage。
  3. 不对obj和showProfileMessage造成任何side effects副作用。

让我们来实现一下call：

第一步：实现 1. 改变this指向

Function.prototype.myCall = function(someThis){
    someThis.fn = this; 
    someThis.fn()
}
showProfileMessage.myCall(obj,'welcome')

showProfileMessage是一个函数对象，我们调用了它原型上的方法myCall，这个方法的this指向的是showProfileMessage,我们在myCall这个方法里将showProfileMessage的指向this添加到了obj上。之后我们再调用obj上的这个showProfileMessage。

# eval()

在实现第二步前，我们先了解一下eval()这个方法。

eval接收一个字符串并把它当做Javascript代码运行。

第二步 接收参数

Function.prototype.myCall(someThis,...args){
    let someThis = someThis || window;
    let fn = Symbol('fn'); //symbol不会与任何别的变量相等
    someThis.fn = this; //将方法的添加到新的context上
    
    let result = eval("someThis.fn(...args)");
    
    delete someThis.fn //删除上的这个属性。避免side effects
    return result;
}

# apply实现

call和apply大同小异，两个方法的区别在于传入参数的形式不同。

call的参数是一个一个传入的。

apply的参数是一个数组。

Function.prototype.myApply = function(someThis,args){
    let someThis = someThis || window;
    let fn = Symbol('fn');
    someThis.fn = this;
    
    let result = eval("someThis.fn(...args)")
    delete someThis.fn; //delete fn property from someThis in order to avoid side effects
    return result;
}

# bind实现

bind没有立即执行改变了上下文的函数，而是：

  1. 对于普通函数，绑定this指向
  2. 对于构造函数，要保证原函数的原型对象上的属性不能丢失。
Function.prototype.bind = function(context,...args){ //和call一样的传参方法
    let self = this;
    
    let fbound = function(){
        self.call(context,args.concat(Array.prototype.slice.call(arguments))) //将bind阶段传入的参数和后面调用bind返回的方法传入的参数一起作为之后调用方法的参数
    }
    
    //如果原函数是构造函数的话，保证原函数的原型属性不丢失。
    fbound.prototype = Object.create(self.prototype);
    
    return fbound
}
