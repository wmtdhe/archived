# 1. 术语

  • 1.1. promise是一个有then方法的obj或者function。
  • 1.2. thenable是定义了then方法的一个obj或者function。
  • 1.3. value是任意JS合法的取值，包含undefined，thenable或者promise。
  • 1.4. exception是用throw statement抛出的一个值。
  • 1.5. reason是一个表明为什么promise被rejected的值。

# 2. 要求

# 2.1. Promise状态

一个promise必须处于以下三个状态中的一个：pending, fulfilled, 或者rejected。

  • 2.1.1. 当处于pending时，一个promise：
    • 2.1.1.1. 可能会过渡到fulfilled或者rejected状态。
  • 2.1.2. 当fulfilled时，一个promise：
    • 2.1.2.1. 不能过渡到别的状态。
    • 2.1.2.2. 必须有一个不能改变value。
  • 2.1.3. 当rejected时，一个promise：
    • 2.1.3.1. 不能过渡到别的状态。
    • 2.1.3.2. 必须有一个不能改变的reason。

# 2.2. then方法

一个promise必须有一个then方法来获取它的当前或者最终的value或者reason。

一个promise的then方法接收两个参数:

promise.then(onFulfilled, onRejected)
  • 2.2.1. onFulfilled，onRejected都是optional的参数：
    • 2.2.1.1. 如果onFulfilled不是一个方法必须被忽略。
    • 2.2.1.2. onRejected不是一个方法必须被忽略。
  • 2.2.2. 如果onFulfilled是一个方法：
    • 2.2.2.1. 当promise被满足时，它必须被调用。 promise的值作为它的第一个参数。
    • 2.2.2.2. 必不能在promise被满足前被调用。
    • 2.2.2.3. 不能被调用超过一次。
  • 2.2.3. 如果onRejected是一个方法:
    • 2.2.3.1. 当promise被拒绝时，它必须被调用。promise的理由作为它的第一个参数。
    • 2.2.3.2. 必不能在promise被拒绝前调用。
    • 2.2.3.3. 不能被调用超过一次。
  • 2.2.4. onFulfilled或者onRejected不能被调用直到execution context栈只有platform code. [3.1]
  • 2.2.5. onFulfilled和onRejected必须被当作方法调用（没有 this参与）。[3.2]
  • 2.2.6. then可以在同一个promise上被多次调用。
    • 2.2.6.1. 如果/当promise被满足，所有相对的 onFulfilled的回调都必须按照它们产生的顺序被执行。
    • 2.2.6.2. 如果/当promise被拒绝，所有相对的 onRejected的回调都必须按照它们产生的顺序被执行。
  • 2.2.7. then必须返回一个promise [3.3].
    promise2 = promise1.then(onFufilled, onRejected);
    • 2.2.7.1. 如果onFulfilled或者 onRejected返回一个值x，执行Promise Resolution Procedure [[Resolve]](promise2, x) 。
    • 2.2.7.2. 如果onFulfilled或者 onRejected抛出一个例外e， promise2必须用这个e作为理由被拒绝。
    • 2.2.7.3. 如果onFulfilled不是一个方法并且 promise1被满足，promise2必须用和 promise1一样的值被满足。
    • 2.2.7.4. 如果onRejected不是一个方法并且 promise1被拒绝，promise2必须用和 promise1一样的理由被拒绝。

# 2.3. The Promise Resolution Procedure

Promise resolution procedure 是一个抽象的操作，以一个promise和一个value作为input - [[Resolve]](promise,x)。如果x是thenable，它会尝试让promise采纳这个x的状态，在基于x的行为和promise一样的猜想下。否则它将用这个x作为值满足promise。

[[Resolve]](promise, x)步骤：

  • 2.3.1. 如果promise和x指的是同一对象，用TypeError作为理由拒绝这个promise。
  • 2.3.2. 如果x是一个promise，采纳它的状态 [3.4]：
    • 2.3.2.1. 如果x在pending状态，promise必须保持pending直到x被满足或被拒绝。
    • 2.3.2.2. 如果/当x被满足，用同样的值满足promise
    • 2.3.2.3. 如果/当x被拒绝，用同样的理由拒绝promise。
  • 2.3.3. 如果x是对象或者方法：
    • 2.3.3.1. then = x.then[3.5]。
    • 2.3.3.2. 如果获取x.then报错e，用这个e拒绝promise。
    • 2.3.3.3. 如果then是一个方法，用x作为上下文调用它，第一个参数是resolvePromise, 第二个参数是rejectPromise:
      • 2.3.3.3.1. 如果/当resolvePromise同一个值y一起被调用，执行[[Resolve]](promise,y)
      • 2.3.3.3.2. 如果/当 rejectPromise同一个理由r被调用，用r拒绝promise。
      • 2.3.3.3.3. 如果resolvePromise,rejectPromise都被调用，或者参数被多次调用，第一次调用执行，之后的调用都被忽略。
      • 2.3.3.3.4. 如果调用then抛错e：
        • 2.3.3.3.4.1. 如果resolvePromise或rejectPromise已经被调用，忽略这个e。
        • 2.3.3.3.4.2. 否则拒绝promise with e。
    • 2.3.3.4. 如果then不是方法，用x满足promise。
  • 2.3.4. 如果x不是对象也不是方法，用x满足promise。

If a promise is resolved with a thenable that participates in a circular thenable chain, such that the recursive nature of [[Resolve]](promise, thenable) eventually causes [[Resolve]](promise, thenable) to be called again, following the above algorithm will lead to infinite recursion. Implementations are encouraged, but not required, to detect such recursion and reject promise with an informative TypeError as the reason. [3.6]
