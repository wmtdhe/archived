# Tree Shaking Intro

JS中用于dead-code的消灭。Tree Shaking 是依赖于ES2015模块语法的静态结构。

# Example

首先在src/math.js目录下导出两个function。

src/math.js

export function square(){
    return x * x;
}

export function cube(x){
    return x * x * x;
}

src/index.js

import { cude } from './math.js'

function component(){
    const element = document.createElement('pre');
    
    element.innnerHTML = ['Hello webpack!',
    '5 cubed is equal to ' + cube(5)
    ].join('\n\n');
    
    return element;
}

document.body.appendChild(component());

webpack.config.js

const path = require('path');

module.exports = {
  entry: './src/index.js',
  output: {
    filename: 'bundle.js',
    path: path.resolve(__dirname, 'dist'),
  },
+ mode: 'development',
+ optimization: {
+   usedExports: true, //<---tree shaking
+ },
};

在上述index.js中我们并没有importmath.js中的square方法，这个square就是所谓的dead code，应该被drop掉。

# 标记一个文件为 side-effect-free

告诉webpack's complier你的code的pureness，这是通过package.json中的sideEffects属性实现的。

{
    "name":"package name",
    "sideEffects":false
}

之前上述的code都没有side effects，所以我们可以直接将sideEffectsmark为false来告诉webpack它可以安全地进行unused export梳理。

A "side effect" is defined as code that performs a special behavior when imported, other than exposing one or more exports. An example of this are polyfills, which affect the global scope and usually do not provide an export.

side effect指的是当被import引入时会执行一些特别的行为，而不是单单暴露一个或多个exports导出。比如说polyfills这种影响全局global scope的通常没有导出。

如果你的code有多个side effects的话，可以通过array来指明：

{
  "name": "your-project",
  "sideEffects": [
    "./src/some-side-effectful-file.js"
  ]
}

Note:

任何引入的文件都会受到tree shaking影响。这意味着如果你用了css-loader来引入css文件的话，这些文件需要被加到sideEffects的array中从而避免在production mode中被drop掉.

# Tree Shaking总结

如果需要利用tree shaking，你需要：

  • 用ES2015 module syntax(i.e. import和export)
  • 保证没有compiler transform ES2015 moudle syntax 到 CommenJS modules
  • 在package.json中添加sideEffects属性
  • 用production mode configuration option来使得多种optimization得以生效。
