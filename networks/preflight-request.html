    # CORS中的预检请求 Preflighted Requests in CORS

    一个CORS预检请求指的是一个检查CORS协议是否被理解，并且服务端知晓请求的方法和请求头的一个请求。

    一个预检(OPTIONS)请求，会携带三个指定的请求头：Access-Control-Request-Method, Access-Control-Request-Headers和Origin

    预检请求通常是浏览器自动发出的

    以下列请求信息为例，在跨域资源请求中（CORS），会先发一个方法为OPTIONS的预请求，然后服务端再响应是否接受这个请求。Access-Control-Request-Method是预请求中的一部分，它会告知服务端之后实际请求所用的方法，在下列例子中会通知服务端接下来的实际请求方法是POST。Access-Control-Request-Header是预请求中告知服务端哪些Header之后会被携带在请求中，在下列例子中会告知接下来的实际请求会携带X-PINGOTHER和Content-Type。在接收到这个预请求的信息后，服务端现在有了一个决定要不要接受在这些条件下的一个请求的机会。

    OPTIONS /resources/post-here/ HTTP/1.1 
Host: bar.other 
Accept: text/html,application/xhtml+xml,application/xml;q=0.9,*/*;q=0.8 
Accept-Language: en-us,en;q=0.5 
Accept-Encoding: gzip,deflate 
Accept-Charset: ISO-8859-1,utf-8;q=0.7,*;q=0.7 
Connection: keep-alive 
Origin: http://foo.example 
Access-Control-Request-Method: POST 
Access-Control-Request-Headers: X-PINGOTHER, Content-Type

    服务端响应时，会携带Access-Control-Allow-Methods，以下列例子来说，服务端允许POST, GET和OPTIONS来进行请求。

    HTTP/1.1 204 No Content
Date: Mon, 01 Dec 2008 01:15:39 GMT 
Server: Apache/2.0.61 (Unix) 
Access-Control-Allow-Origin: http://foo.example 
Access-Control-Allow-Methods: POST, GET, OPTIONS 
Access-Control-Allow-Headers: X-PINGOTHER, Content-Type 
Access-Control-Max-Age: 86400 
Vary: Accept-Encoding, Origin
Keep-Alive: timeout=2, max=100 
Connection: Keep-Alive

    # 简单请求 (Simple Request)

    简单请求不会触发预检请求

    一个简单请求需要满足以下条件：

    • 是以下的某一个方法
      • GET
      • HEAD
      • POST
    • 除开那些自动被浏览器设置的请求头（例如，Connection, User-Agent），剩下的能人为设置的请求头只能是如下被列在CORS-safelisted request-header中的请求头：
      • Accept
      • Accept-Language
      • Content-Language
      • Content-Type(注意接下来的附加要求)
      • DPR
      • Downlink
      • Save-Data
      • Viewport-Width
      • Width
    • Content-Type请求头只能是以下值：
      • application/x-www-form-urlencoded
      • multiplart/form-data
      • text/plain
    • 请求中使用到的XMLHttpRequestUpload对象没有注册事件监听
    • 请求中没有使用ReadableStream对象
