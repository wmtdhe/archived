    状态码被分为五类：

    1. 1XX：表示目前是协议处理的中间状态，还要后续操作
    2. 2XX：表示成功状态
    3. 3XX：重定向状态，资源位置发生变动，需要重新请求
    4. 4XX：请求报文有误
    5. 5XX：服务端发生错误

    # 1XX

    101 Switching Protocols 在HTTP升级为WebSocket的时候，如果服务器同意变更，就会发送状态码101。

    # 2XX

    200 OK 见得最多的状态码，通常在响应体放有数据。

    204 No Content 含义与200相同，但响应头后没有body数据。

    206 Partial Content 表示部分内容，它的使用场景为HTTP分块下载和断点续传，当然也会带上相应的响应头字段Content-Range。

    # 3XX

    301 Moved Permanently 永久重定向，应用场景：当协议从HTTP升级到了HTTPS，应当返回301，这时候浏览器默认会做缓存优化，第二次访问的时候自动访问重定向的那个地址。

    302 Moved Temporarily 暂时重定向，浏览器不做缓存优化，下次访问的时候也先访问原来的地址再重定向到新地址。

    304 Not Modified 协商缓存命中时返回的状态。

    # 4XX

    400 Bad Request 笼统的错误提示

    403 Forbidden 服务器禁止访问，原因很多，比如法律，信息敏感等。

    404 Not Found 资源没有找到，表示服务器上没有对应的资源。

    405 Method Not Allowed 请求方法不被服务器允许

    406 Not Acceptable 资源无法满足客服端条件

    408 Request Timeout 服务器等待了太长时间。

    409 Conflict 多个请求发生了冲突

    413 Request Entity Too Large 请求体数据过大

    414 Request-URI Too Long 请求行里的URI太大

    429 Too Many Request 客户端发送太多请求

    431 Request Header Fields Too Large 请求头的字段内容太大

    # 5XX

    500 Internal Server Error 仅仅告诉你服务端出错，没说哪里错

    501 Not Implemented 表示客户端的请求的功能还不支持

    502 Bad Gateway 服务器本身是正常的，但访问的时候出错了

    503 Service Unavailable 表示服务器忙，暂时无法响应服务
